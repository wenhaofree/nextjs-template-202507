## Prerequisites

Before you begin, make sure you have the following installed on your system:

- **Node.js** (version 18.0 or higher)
- **PNPM** package manager (recommended for this project)
- **Git** for version control
- A code editor (we recommend **VS Code**)
- **PostgreSQL** database (we recommend **Neon** for cloud hosting)

## Quick Setup

### 1. Clone the Repository

```bash
git clone <your-shipsaas-repository-url>
cd nextjs-demo
```

### 2. Install Dependencies

This project uses **PNPM** as the package manager for optimal performance:

```bash
# Install PNPM if you haven't already
npm install -g pnpm

# Install project dependencies
pnpm install
```

### 3. Environment Configuration

Copy the example environment file and configure your settings:

```bash
cp .env.example .env.local
```

Edit `.env.local` with your configuration. Here are the essential settings:

```bash
# -----------------------------------------------------------------------------
# Basic Configuration (Required)
# -----------------------------------------------------------------------------
NEXT_PUBLIC_WEB_URL=http://localhost:3000
NEXT_PUBLIC_PROJECT_NAME=ShipSaaS

# -----------------------------------------------------------------------------
# Authentication (Required)
# Generate AUTH_SECRET with: openssl rand -base64 32
# -----------------------------------------------------------------------------
AUTH_SECRET=your-generated-auth-secret

# Google OAuth (Optional)
AUTH_GOOGLE_ID=your-google-client-id
AUTH_GOOGLE_SECRET=your-google-client-secret
NEXT_PUBLIC_AUTH_GOOGLE_ENABLED=true
NEXT_PUBLIC_AUTH_GOOGLE_ONE_TAP_ENABLED=false

# GitHub OAuth (Optional)
AUTH_GITHUB_ID=your-github-client-id
AUTH_GITHUB_SECRET=your-github-client-secret
NEXT_PUBLIC_AUTH_GITHUB_ENABLED=true

# -----------------------------------------------------------------------------
# Database (Required)
# -----------------------------------------------------------------------------
DATABASE_URL=your-postgresql-database-url

# -----------------------------------------------------------------------------
# Stripe Payment (Required for payments)
# -----------------------------------------------------------------------------
STRIPE_PUBLIC_KEY=your-stripe-publishable-key
STRIPE_PRIVATE_KEY=your-stripe-secret-key
STRIPE_WEBHOOK_SECRET=your-stripe-webhook-secret

# -----------------------------------------------------------------------------
# Email Service (Optional)
# -----------------------------------------------------------------------------
RESEND_API_KEY=your-resend-api-key
NEXT_PUBLIC_FROM_EMAIL=noreply@yourdomain.com
```

### 4. Database Setup

Set up your database schema using Prisma:

```bash
# Generate Prisma client
pnpm db:generate

# Push schema to database
pnpm db:push

# Optional: Seed database with initial data
pnpm db:seed
```

### 5. Start Development Server

Run the development server with Turbopack for optimal performance:

```bash
pnpm dev
```

The server will start on [http://localhost:3000](http://localhost:3000). Turbopack is automatically enabled for development mode.

## What's Next?

Now that you have ShipSaaS running locally, you can:

1. **Explore the Features** - Navigate through the application to see all available features
   - AI Chat interface powered by ChatGPT
   - AI Image Generation capabilities
   - User authentication and dashboard
   - Stripe payment integration
   - Multi-language support (English/Chinese)

2. **Customize the Design** - Modify components and styling to match your brand
   - Update theme colors in `src/app/globals.css`
   - Customize components in `src/components/ui/`
   - Modify landing page sections in `src/components/sections/`

3. **Configure Services** - Set up your external services
   - Configure OAuth providers (Google, GitHub)
   - Set up Stripe for payments and subscriptions
   - Configure email service with Resend
   - Set up AI services (OpenAI API)

4. **Deploy Your App** - Follow our deployment guide to go live
   - Deploy to Vercel for optimal Next.js performance
   - Configure production environment variables
   - Set up domain and SSL certificates

## Troubleshooting

### Common Issues

**Port already in use:**
```bash
# Kill the process using port 3000 (macOS/Linux)
lsof -ti:3000 | xargs kill -9

# Or use a different port
pnpm dev -- --port 3001
```

**Database connection issues:**
- Verify your `DATABASE_URL` in `.env.local`
- Ensure your PostgreSQL/Neon database is accessible
- Run `pnpm db:push` to sync the schema
- Check database credentials and network connectivity

**Build errors:**
- Clear Next.js cache: `rm -rf .next`
- Reinstall dependencies: `rm -rf node_modules && pnpm install`
- Check for TypeScript errors: `pnpm build`

**PNPM issues:**
- Update PNPM: `npm install -g pnpm@latest`
- Clear PNPM cache: `pnpm store prune`
- Verify Node.js version: `node --version` (should be 18+)

**Environment variable issues:**
- Ensure `.env.local` exists and has correct values
- Restart development server after changing environment variables
- Check for typos in variable names
