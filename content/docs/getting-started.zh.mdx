## 前置要求

开始之前，请确保您的系统已安装以下软件：

- **Node.js**（版本 18.0 或更高）
- **PNPM** 包管理器（本项目推荐使用）
- **Git** 版本控制工具
- 代码编辑器（推荐使用 **VS Code**）
- **PostgreSQL** 数据库（推荐使用 **Neon** 云托管）

## 快速设置

### 1. 克隆仓库

```bash
git clone <your-shipsaas-repository-url>
cd nextjs-demo
```

### 2. 安装依赖

本项目使用 **PNPM** 作为包管理器以获得最佳性能：

```bash
# 如果尚未安装 PNPM
npm install -g pnpm

# 安装项目依赖
pnpm install
```

### 3. 环境配置

复制示例环境文件并配置您的设置：

```bash
cp .env.example .env.local
```

编辑 `.env.local` 文件，配置您的设置。以下是必要的配置：

```bash
# -----------------------------------------------------------------------------
# 基础配置（必填）
# -----------------------------------------------------------------------------
NEXT_PUBLIC_WEB_URL=http://localhost:3000
NEXT_PUBLIC_PROJECT_NAME=ShipSaaS

# -----------------------------------------------------------------------------
# 身份验证（必填）
# 使用以下命令生成 AUTH_SECRET：openssl rand -base64 32
# -----------------------------------------------------------------------------
AUTH_SECRET=your-generated-auth-secret

# Google OAuth（可选）
AUTH_GOOGLE_ID=your-google-client-id
AUTH_GOOGLE_SECRET=your-google-client-secret
NEXT_PUBLIC_AUTH_GOOGLE_ENABLED=true
NEXT_PUBLIC_AUTH_GOOGLE_ONE_TAP_ENABLED=false

# GitHub OAuth（可选）
AUTH_GITHUB_ID=your-github-client-id
AUTH_GITHUB_SECRET=your-github-client-secret
NEXT_PUBLIC_AUTH_GITHUB_ENABLED=true

# -----------------------------------------------------------------------------
# 数据库（必填）
# -----------------------------------------------------------------------------
DATABASE_URL=your-postgresql-database-url

# -----------------------------------------------------------------------------
# Stripe 支付（支付功能必填）
# -----------------------------------------------------------------------------
STRIPE_PUBLIC_KEY=your-stripe-publishable-key
STRIPE_PRIVATE_KEY=your-stripe-secret-key
STRIPE_WEBHOOK_SECRET=your-stripe-webhook-secret

# -----------------------------------------------------------------------------
# 邮件服务（可选）
# -----------------------------------------------------------------------------
RESEND_API_KEY=your-resend-api-key
NEXT_PUBLIC_FROM_EMAIL=noreply@yourdomain.com
```

### 4. 数据库设置

使用 Prisma 设置您的数据库架构：

```bash
# 生成 Prisma 客户端
pnpm db:generate

# 将架构推送到数据库
pnpm db:push

# 可选：使用初始数据填充数据库
pnpm db:seed
```

### 5. 启动开发服务器

使用 Turbopack 运行开发服务器以获得最佳性能：

```bash
pnpm dev
```

服务器将在 [http://localhost:3000](http://localhost:3000) 启动。开发模式下会自动启用 Turbopack。

```
│   │   ├── dashboard/       # 仪表板组件
│   │   └── magicui/         # 增强 UI 组件
│   ├── lib/                 # 工具函数和配置
│   ├── hooks/               # 自定义 React 钩子
│   ├── i18n/                # 国际化设置
│   └── types/               # TypeScript 类型定义
├── content/
│   ├── docs/                # 文档内容（MDX）
│   └── blog/                # 博客内容（MDX）

```

## 可用命令

### 开发命令
```bash
pnpm dev          # 使用 Turbopack 启动开发服务器
pnpm build        # 构建生产版本
pnpm start        # 启动生产服务器
pnpm lint         # 运行 ESLint
```

### 数据库命令
```bash
pnpm db:generate  # 生成 Prisma 客户端
pnpm db:push      # 将架构更改推送到数据库
pnpm db:migrate   # 运行数据库迁移
pnpm db:studio    # 打开 Prisma Studio
pnpm db:seed      # 使用初始数据填充数据库
```

## 故障排除

### 常见问题

**端口已被占用：**
```bash
# 终止使用端口 3000 的进程（macOS/Linux）
lsof -ti:3000 | xargs kill -9

# 或使用不同端口
pnpm dev -- --port 3001
```

**数据库连接问题：**
- 验证 `.env.local` 中的 `DATABASE_URL`
- 确保您的 PostgreSQL/Neon 数据库可访问
- 运行 `pnpm db:push` 同步架构
- 检查数据库凭据和网络连接

**构建错误：**
- 清除 Next.js 缓存：`rm -rf .next`
- 重新安装依赖：`rm -rf node_modules && pnpm install`
- 检查 TypeScript 错误：`pnpm build`

**PNPM 问题：**
- 更新 PNPM：`npm install -g pnpm@latest`
- 清除 PNPM 缓存：`pnpm store prune`
- 验证 Node.js 版本：`node --version`（应为 18+）

**环境变量问题：**
- 确保 `.env.local` 存在且具有正确的值
- 更改环境变量后重启开发服务器
- 检查变量名称中的拼写错误
