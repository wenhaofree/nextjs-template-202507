## 前置要求

开始之前，请确保您的系统已安装以下软件：

- **Node.js**（版本 18.0 或更高）
- **npm**、**yarn** 或 **pnpm** 包管理器
- **Git** 版本控制工具
- 代码编辑器（推荐使用 **VS Code**）

## 快速设置

### 1. 克隆仓库

```bash
git clone https://github.com/ShipSaaSCo/nextjs-template-202507.git
cd nextjs-template-202507
```

### 2. 安装依赖

选择您喜欢的包管理器：

```bash
# 使用 npm
npm install

# 使用 yarn
yarn install

# 使用 pnpm
pnpm install
```

### 3. 环境配置

复制示例环境文件并配置您的设置：

```bash
cp .env.example .env.local
```

编辑 `.env.local` 文件，配置您的设置：

```bash
# 数据库
DATABASE_URL="your-neon-database-url"

# 身份验证
NEXTAUTH_SECRET="your-nextauth-secret"
NEXTAUTH_URL="http://localhost:3000"

# OAuth 提供商
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"

# Stripe
STRIPE_SECRET_KEY="your-stripe-secret-key"
STRIPE_PUBLISHABLE_KEY="your-stripe-publishable-key"
STRIPE_WEBHOOK_SECRET="your-stripe-webhook-secret"

# 邮件
RESEND_API_KEY="your-resend-api-key"
```

### 4. 数据库设置

设置您的数据库架构：

```bash
npx prisma generate
npx prisma db push
```

### 5. 启动开发服务器

使用 Turbopack 运行开发服务器以获得最佳性能：

```bash
npm run dev --turbo
# 或
yarn dev --turbo
# 或
pnpm dev --turbo
```

在浏览器中打开 [http://localhost:3000](http://localhost:3000) 查看您的应用程序运行情况。

## 项目结构

以下是 ShipSaaS 项目中关键目录的概述：

```
nextjs-template-202507/
├── src/
│   ├── app/                 # Next.js App Router 页面
│   ├── components/          # 可重用的 UI 组件
│   ├── lib/                 # 工具函数和配置
│   └── styles/              # 全局样式和 Tailwind 配置
├── content/
│   └── docs/                # 文档内容
├── messages/                # 国际化消息
├── prisma/                  # 数据库架构和迁移
└── public/                  # 静态资源
```

## 故障排除

### 常见问题

**端口已被占用：**
```bash
# 终止使用端口 3000 的进程
lsof -ti:3000 | xargs kill -9
```

**数据库连接问题：**
- 验证 `.env.local` 中的 `DATABASE_URL`
- 确保您的 Neon 数据库可访问
- 运行 `npx prisma db push` 同步架构

**构建错误：**
- 清除 Next.js 缓存：`rm -rf .next`
- 重新安装依赖：`rm -rf node_modules && npm install`

需要更多帮助？查看我们的[环境设置](/docs/environment-setup)指南，获取详细的配置说明。
