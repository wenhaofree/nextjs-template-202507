# ShipSaaS 入门指南

欢迎使用 ShipSaaS！如果您想快速构建和启动 SaaS 应用程序，您来对地方了。ShipSaaS 是一个完整的 Next.js 样板，包含创建盈利 SaaS 业务所需的一切。

## 什么是 ShipSaaS？

ShipSaaS 是一个综合开发平台，提供：

- **🚀 Next.js 15 框架** - 基于最新的 Next.js 和 React 19 构建
- **🔐 身份验证系统** - 使用 NextAuth.js 的完整用户管理
- **💳 支付集成** - Stripe 集成，支持订阅和一次性支付
- **🌍 国际化** - 使用 next-intl 的多语言支持
- **📧 邮件系统** - 新闻通讯和事务性邮件
- **📊 仪表板** - 用户和管理员仪表板
- **📝 博客和文档** - 内置博客和文档系统
- **🎨 UI 组件** - 使用 TailwindCSS 的精美组件
- **🔍 SEO 优化** - 内置 SEO 最佳实践

## 快速开始指南

### 1. 安装

首先，克隆 ShipSaaS 仓库并安装依赖：

```bash
git clone https://github.com/your-repo/shipsaas.git
cd shipsaas
pnpm install
```

### 2. 环境设置

复制环境变量并配置您的设置：

```bash
cp .env.example .env.local
```

使用以下内容更新您的 `.env.local` 文件：

```bash
# 数据库
DATABASE_URL="your-neon-database-url"

# 身份验证
NEXTAUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"

# OAuth 提供商
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# Stripe
STRIPE_SECRET_KEY="your-stripe-secret-key"
STRIPE_PUBLISHABLE_KEY="your-stripe-publishable-key"
STRIPE_WEBHOOK_SECRET="your-webhook-secret"
```

### 3. 数据库设置

ShipSaaS 使用 Neon PostgreSQL 作为数据库。运行迁移：

```bash
pnpm db:migrate
pnpm db:seed
```

### 4. 启动开发

启动开发服务器：

```bash
pnpm dev
```

您的应用程序将在 `http://localhost:3000` 可用。

## 主要功能概述

### 身份验证系统

ShipSaaS 包含完整的身份验证系统：

- **社交登录** - Google 和 GitHub OAuth
- **邮箱/密码** - 传统身份验证
- **密码重置** - 基于邮件的密码恢复
- **用户资料** - 可自定义的用户资料
- **角色管理** - 管理员和用户角色

### 支付集成

Stripe 集成提供：

- **订阅计划** - 多个定价层级
- **一次性支付** - 用于附加功能
- **账单管理** - 客户门户集成
- **Webhook 处理** - 自动订阅更新

### 仪表板系统

用户和管理员仪表板包括：

- **分析** - 使用和收入指标
- **用户管理** - 管理员用户控制
- **账单概览** - 订阅状态
- **设置** - 账户配置

### 添加功能

ShipSaaS 设计为可扩展的。您可以通过以下方式轻松添加新功能：

1. 在 `src/app/api/` 中创建新的 API 路由
2. 在 `src/app/[locale]/` 中添加新页面
3. 在 `src/components/` 中构建自定义组件
4. 在 `src/lib/db/` 中扩展数据库架构

## 部署

### Vercel 部署

ShipSaaS 针对 Vercel 部署进行了优化：


确保在生产环境中设置所有必需的环境变量。

```bash

```
